写在前面

个人博客很早就已经在互联网上出现。google的blog曾是世界上最流行的博客系统，08年北京奥运会期间对网络的封锁非常宽松，在那个时候接触了包含blog在内的一批优秀的互联网服务。现在想起来仍感到嘘唏，那个时候访问youtube毫无压力，twitter上面也有许多国人。
后来随着互联网审查和网络封锁的加剧，许多国外服务都无法正常访问，加上工作的原因，blog就这样荒废了。

选择GitHub Pages

对于Github Pages,官方的描述是这样的：
“Websites for you and your projects, hosted directly from your GitHub repository. Just edit, push, and your changes are live.”
Github Pages的优势在于你无需关心网站是如何运行的，甚至不必关心网站的构建、运营，你所做的就是写文章和发布。
GitHub Pages提供了开放的静态网站搭建和托管平台，这里没有流量限制，提供免费托管服务，300MB的免费空间能够满足大部分的个人博客。借助于Github的架构，在世界各地访问博客的速度都还可以。
本次搭建使用的是Hexo博客框架，Hexo 是一个快速、简洁且高效的博客框架。Hexo 使用Markdown（或其他渲染引擎）解析文章，在几秒内，即可利用靓丽的主题生成静态网页。更重要的是：使用方便，部署快捷。

适用人群

本系列文章并不能满足所有需求，毕竟GitHub Pages的静态网站属性限制了网站的体量，这里没有数据库、没有nginx来做负载均衡，这些都限定了网站只能是小型的、轻量化的技术博客。
如果你的需求在下面列出，那么这个系列的文章就适合你：
1.熟悉基本开发的IT人员。
2.在完成一个项目后希望将自己所遇到的问题和经验分享出来。
3.学习和研究之后希望有一个地方能够记录笔记。
4.没有太多时间和精力去维护和折腾网站，只是希望拥有一个自己的技术博客。

GitHub Pages搭建步骤简述

搭建过程可以分为以下4个步骤：
1.准备本地环境，包括系统环境的准备工作和安装程序的准备工作。
2.各个环节的安装和配置工作。
3.Hexo的配置和博客编写工作。
4.提交至GitHub。
本系列的文章的搭建过程也是按照以上步骤进行。

系列文章规划

这个系列的文章分为若干模块。后续新功能的使用将会逐步写出来。目前就打算写前六个章节，Hexo框架虽然简洁易用，但也有不少功能缺少，后期将会对这些功能逐步完善和改进，如果喜欢这个系列就多来关注，目前博客还没有集成评论功能，如果发现有错误或不明白的地方欢迎到我的GitHub项目下面提交issues进行留言和吐槽，我在不忙时将会进行回复。

建议

如果初次接触GitHub、Hexo、Node.js，建议找个懒洋洋的下午或夜深人静的晚上将一至五章一气呵成，这样可以减少出错的概率，少爬许多坑。
如果不打算使用IntelliJ IDEA，那么前五章完全可以满足搭建一个博客的需求，博客搭建好后通过Markdown编辑工具就可以写博客，实际上文本编辑工具同样也可以写，只是稍微麻烦一些。
