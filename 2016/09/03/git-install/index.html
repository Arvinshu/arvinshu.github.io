Git 安装配置

发表于 2016-09-03

Git是目前最先进的分布式版本控制系统，没有之一。这里简单介绍Git的安装步骤。

在Windows 10 上安装

本次安装环境如下：

操作系统：
Windows 10 Pro 64-bit Operating System

Git版本：
Git-2.9.3-rebase-i-64-bit.exe

访问Git官方网站，根据实际环境下载合适的版本：

下载完成后双击exe程序进行安装，安装过程很简单，第一步阅读GNU License，选择下一步。

选择安装路径，此处一般采用默认路径。在windows较早的版本上常常会将安装路径修改为不带空格的目录，以防止出现各种问题，但是在windows 10上面这个情况得到改善，因此安装在 Program Files目录下也是可以的。

选择要安装的组件，如果需要创建桌面快捷方式、添加右键Git、关联文件等都可以在这一步进行选择和配置。

选择在开始菜单中显示的名字，默认就好。

添加系统环境变量:
第一个选项(Use Git from Git Bash only)表示只在Git Bash中使用Git命令。
第二个选项(Use Git from the Windows Command Prompt)表示可以在Git Bash和Windows CMD中都可以使用Git命令。
默认是第二个选项，不用更改，点击下一步。

选择换行符，Windows系统和Unix系统在处理换行符的方式上有很大不同，这是历史遗留问题，有兴趣的同学可以自行Google，这一步使用默认就好。

选择默认命令行风格，MinTTY和Windows CMD。我比较喜欢MinTTY，命令行的色彩标识看上去很舒服。当然还有一个更重要的原因：MinTTY命令符合Linux的习惯。选择哪个全凭个人喜好。

配置额外的功能，系统缓存和证书管理，建议都勾选上。

开始安装。

安装完成，去掉”View Release Note”,如果你不想打开版本信息的话。

如果出现Git版本信息则说明安装成功

自此，Git在Windows 环境中安装完成。
