GitHub Pages搭建系列(二) - Github账户配置

发表于 2016-09-04

GitHub账户的配置和GitHub Pages仓库创建.

注册GitHub账户

登陆 GitHub,在右上角点击绿色按钮”sign up”进入到注册页面,输入用户名,邮箱,密码,按照3个steps提示做完就注册成功.

##配置证书验证

证书验证介绍

熟悉Liunx的同学都知道,使用ssh连接Linux系统时需要输入用户名和密码,当然也可以配置免密码登陆(验证证书登陆),验证证书登录步骤如下:
1.客户端生成证书:私钥(id_rsa)和公钥(id_rsa.pub),私钥存放在放在客户端.

2.服务器添加信用公钥:上传客户端生成的公钥(id_rsa.pub)至ssh服务器,将公钥添加到指定的文件中完成ssh证书登录的配置.

GitHub的证书登陆使用了同样的原理:用户在本地成好私钥和公钥,将公钥的信息填写至账户进行登记,登记完成后用户就可以使用本地私钥连接自己的GitHub账户提交,管理代码.私钥非常重要,如果设置和保持不当,那么拥有私钥的任何人都有权限访问你的账户,因此要做好私钥管理工作,不要轻易泄露出去.

创建和配置仓库

###创建仓库
登陆GitHub,点击”New repository”

GitHub Pages是一个仓库(Repository),但是它比较特殊,仓库名字是固定格式:
你GitHub用户名.github.io
比如我的GitHub名字是arvinshu(用户名不区分大小写),那么我GitHub Pages的的仓库名就必须是arvinshu.github.io .如果你没有自己的域名只打算使用GitHub Pages提供的域名,那么请不要尝试使用其他名字,否则爬过坑的我将不会告诉你在第五章提交博客后浏览器出现 “404 not found”的原因.
Description是可选项,填写项目描述.
最后记得勾选上”Initialize this repository with a README”,初始化README.

创建成功后还需最后一个步骤:自动生成测试页面.点击仓库的”Settings”.

找到GitHub Pages下的”Launch automatic page generator”.

接下来进入测试web的编辑页面,由于只是用来测试,这里就使用默认好了.直接点击”Continue to layouts”.

在上方找一个喜欢的模板样式,选择”Public page”.

回到仓库,这时仓库里就多了若干文件和目录.

测试

GitHub的配置到此就完成了,此时可以尝试访问你的测试page,能够打开说明配置正确.如果测试时候显示404,那就先做下一步的工作,GitHub Pages创建完成后需要等待一段时间才能访问,大概20分钟吧.但是在配置完成了1个小时后仍然打不开,那就需要检查配置了.
